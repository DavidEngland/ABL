# ABL — Curvature‑Aware MOST & Richardson‑Number Toolkit

Purpose
- Repository of code, notes and notebooks to explore curvature of the gradient Richardson number (Ri_g), Jensen bias, and grid-aware corrections for SBL parameterizations.
- Audience: researchers (McNider, Biazar), model developers, and students.

Quick start (Colab)
- Open the interactive notebook in Google Colab (replace USER/REPO):
  `https://colab.research.google.com/github/USERNAME/REPO/blob/main/notebooks/Curvature_Demo.ipynb`
- In Colab, run the first cell to install minimal requirements.

Quick start (local)
1. Create environment:
   - python 3.10+ recommended
   - `python -m venv .venv && source .venv/bin/activate`
2. Install dependencies:
   - `pip install -r requirements.txt`
3. Launch:
   - `jupyter lab` or `jupyter notebook`
   - Open `notebooks/Curvature_Demo.ipynb`

Notebooks
- Primary demo: `notebooks/Curvature_Demo.ipynb` — interactive sliders, analytic forms, and synthetic NetCDF generator.
- See `notebooks/README.md` for Colab badge template and running notes.

Development notes & pre‑push checklist
- Run unit / smoke checks:
  - Open `notebooks/Curvature_Demo.ipynb` and run key cells (core functions cell, demo cell, synthetic NetCDF cell).
- Verify requirements: ensure `requirements.txt` installs in a fresh env.
- Linting / formatting (optional):
  - `flake8` or `ruff` on `*.py` files.
- Commit & push workflow:
  - Create feature branch: `git checkout -b feat/curvature-fix`
  - Stage and commit: `git add -A && git commit -m "Update README and notebooks run instructions"`
  - Push: `git push origin feat/curvature-fix`
  - Open PR, assign reviewers (McNider, Biazar), merge when approved.
